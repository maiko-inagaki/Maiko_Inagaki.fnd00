プログラミングの基礎

基礎演習

以下の3つを自分の言葉で定義してみてください。

expression（式）

値を導き出すためのひとまとまりのコードのこと

operator（演算子）

値を導き出すために必要なもの
算術演算子 + * など
比較演算子 ==　< など
倫理演算子　and　OR など

type（データ型）

データのカテゴリのこと
（数値型number、文字列型string、真偽値型booleanなど）

【number（数値型）】

1.Google Chrome のコンソールに 1 行ずつ、以下の式を入力してみましょう。それぞれ何が起こるでしょうか。

4 + 10;　=> 10
1 * 3; => 3
12 * 4; => 3
4 % 2; => 0
5 % 2; => 1
5 / 1 - 99; => -94
5000 * -100 * (1 + 2) * (5 * 6); => -45000000
1241 / 9 + 99; => 236.88888888888889

※式の終わりはすべてセミコロンで終了していること

2.% 演算子は何をしますか？

割り算の余り

40 * 365 * 24 * 60 => 21024000

string（文字列型）

1.コンソール上にて、自分の名前を string（文字列型） で入力してみましょう。

"maiko inagaki"

2.+ 演算子を使って、複数の文字列を concatenate （結合）してみましょう。

"super" + " beaber" => 'super beaber'

やってみよう！

1.文字列型（string）の値 と 数値型（number）の値 を足したらどうなりますか？

"abc" + 150 => 'abc150'

2.その順番を逆にしたらどうなりますか (例 数値型（number） + 文字列型（string）)？

150 + "abc" => '150abc'

3.数値型（number） の 5 に 文字列型（string） の "5" を掛け算するとどうなりますか？

5 * "5" => 25

4.コンソールに 1 行ずつ、以下の式を入力してみましょう。.length を追加するとどうなると思いますか？ ほかの文字列の長さも確認してみましょう。

"hello".length; => 5
"hello world".length; => 11
"123".length; => 3
"hello".toUpperCase(); => 'HELLO' 大文字に変わった
"HELLO".toLowerCase();　=> 'hello' 小文字に変わった
"hello".endsWith("o");　=> true 文字列の後方ががoかどうか真偽を調べた
"hello".endsWith("e");　=> false 文字列の後方がeかどうか審議を調べた
"hello".endsWith("llo");　=>　true　文字列の後方がlloで終わっているか真偽を調べた
"hello".endsWith();　=>false 文字列の後方の文字を知らべるメソッドだか引数が空白
